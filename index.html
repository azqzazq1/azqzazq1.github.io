About

I am an independent offensive security researcher focused on the Linux kernel, eBPF subsystem, and container isolation boundaries. My work centers on finding architectural blind spots in defensive security mechanisms, places where the security framework is correct, but the kernel never consults it.

My primary research investigates the trust assumptions embedded in Linux Security Modules (AppArmor, SELinux), eBPF-based runtime monitors (Falco, Tracee, Tetragon), and container orchestration platforms (Kubernetes). Rather than breaking cryptographic primitives or exploiting memory corruption, I focus on logic-level gaps: code paths that exist in the kernel but were never wired into the security framework, telemetry pipelines that can be silently poisoned, and isolation boundaries that hold on paper but fail under specific syscall sequences.

Research Interests

Linux kernel eBPF LSM framework AppArmor SELinux Kubernetes container isolation io_uring telemetry deception
  • Defensive system gap analysis. Systematic discovery of kernel subsystems that perform security-sensitive operations without consulting LSM hooks. The security policy is enforced correctly, but the problem is that the kernel has code paths that never ask.
  • eBPF security boundaries. Investigating both the offensive capabilities of eBPF (pre-check manipulation, post-syscall buffer rewriting) and the defensive fragility of eBPF-based monitors (map state integrity, program array trust).
  • Container and orchestrator isolation. Exploring how Kubernetes RBAC, admission controllers, node authorizers, and network policies hold up under adversarial conditions, and where they don't.
  • Telemetry trust model. The gap between what actually happened on a system and what security agents observe and report. If the agent reads data, but the data no longer represents the event, the telemetry pipeline has a fundamental trust problem.

SunnyDayBPF

telemetry deception | eBPF kprobe/kretprobe | post syscall

Post syscall observation layer deception via eBPF. Hooks read-like syscalls and rewrites data in user space buffers after the kernel writes it but before security agents parse it. The event still happens. The agent still reads data. But the data no longer represents the original event.

SunnyMapBPF

BPF map poisoning | Falco, Tracee, Tetragon | runtime monitor evasion

BPF map state poisoning against the three major eBPF runtime security monitors. A same-privilege process can overwrite policy maps, program arrays, and event routing tables to achieve complete telemetry suppression with zero errors, zero health check failures, and immediate effect.
