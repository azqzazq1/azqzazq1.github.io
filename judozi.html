Overview

Judozi is a modular offensive security framework written in pure Go. It compiles to a single static binary with zero external dependencies, designed for environments where installing packages or toolchains is not an option. The framework uses a plugin-based module system where each module implements a common interface and is auto-discovered at startup.

The framework currently ships with three modules: kernel (automated privilege escalation), persistence (post-exploitation persistence mechanisms), and recon (system enumeration and reconnaissance). After successful privilege escalation, Judozi drops into an interactive shell session with full module access.

Architecture

Every module implements the Module interface: Name(), Description(), Category(), and Run(args []string). The registry auto-discovers registered modules and dispatches execution based on CLI arguments or interactive selection. After a successful kernel exploit, the framework automatically transitions into an interactive shell with access to all modules.

Module Interface

// Every Judozi module implements this interface
type Module interface {
    Name()        string
    Description() string
    Category()    string
    Run(args []string) error
}

Kernel Module: Execution Pipeline

The kernel module is the primary attack module. It automates the entire privilege escalation workflow from kernel version detection through exploit selection, compilation or binary download, and execution.

Real World Testing

Judozi has been tested in production cloud environments, including Azure Container instances running kernel 5.15.0. The binary fallback system was validated in GCC-free environments: the framework detected 10 matching exploits, automatically downloaded precompiled binaries, and executed them without requiring any compiler toolchain on the target.

nextjs@container:/tmp$ uname -r

nextjs@container:/tmp$ ./judozi kernel -auto
